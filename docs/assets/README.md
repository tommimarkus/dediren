# Docs assets

## `pipeline.svg` — the pipeline diagram

`pipeline.svg` is **generated by Dediren from its own pipeline**, not drawn by
hand — the README diagram dogfoods the tool. The inputs live beside it:

- [`pipeline.source.json`](pipeline.source.json) — the semantic source model.
- [`pipeline.render-policy.json`](pipeline.render-policy.json) — the render policy.

Regenerate it from a built bundle (the glob picks the newest bundle under
`dist/`, so it is version-agnostic):

```bash
BUNDLE=$(ls -d dist/dediren-agent-bundle-* | grep -v '\.tar\.gz$' | sort | tail -1)

"$BUNDLE/bin/dediren" project --target layout-request --plugin generic-graph \
  --view pipeline --input docs/assets/pipeline.source.json > /tmp/pipeline-req.json

"$BUNDLE/bin/dediren" layout --plugin elk-layout \
  --input /tmp/pipeline-req.json > /tmp/pipeline-res.json

"$BUNDLE/bin/dediren" render --plugin render \
  --policy docs/assets/pipeline.render-policy.json --input /tmp/pipeline-res.json \
  | jq -r '.data.artifacts[] | select(.artifact_kind=="svg") | .content' \
  > docs/assets/pipeline.svg
```

If `dist/` is empty, build a bundle first with
`./mvnw -pl dist-tool -am verify -Pdist-build`.
